Память:

    Физическая память:

        free -h - Показывает использование физической памяти и swap в удобном для восприятия формате.

        vmstat - Предоставляет статистику по использованию памяти, процессов, дисков и т.д.

    Виртуальная память:

        pmap -x <PID> - Показывает карту памяти процесса, включая использование виртуальной памяти.

        cat /proc/<PID>/maps - Аналогично pmap, но в текстовом формате.


Сетевые интерфейсы

Просмотр сетевых интерфейсов:

    ip a или ip addr - Показывает информацию о всех сетевых интерфейсах, включая IP-адреса, MAC-адреса и состояние интерфейсов.

    ifconfig - Старый инструмент для просмотра и настройки сетевых интерфейсов.

    nmcli device show - Показывает информацию о сетевых устройствах, используя NetworkManager.

Просмотр открытых портов

Использование netstat:

    netstat -tuln - Показывает открытые TCP и UDP порты, а также слушающие сокеты.

        -t: Отображает только TCP порты.

        -u: Отображает только UDP порты.

        -l: Отображает только слушающие сокеты.

        -n: Отображает IP-адреса и номера портов в числовом формате.

ps aux - Показывает все запущенные процессы в системе.

        a: Отображает процессы всех пользователей.

        u: Отображает подробную информацию о процессах.

        x: Отображает процессы, которые не привязаны к терминалу.

Использование top:

    top - Интерактивный монитор системы, показывающий запущенные процессы и использование ресурсов в реальном времени.

Использование htop:

    htop - Улучшенная версия top с более удобным интерфейсом.

    ps aux | grep 'Z' - Фильтрует процессы, у которых состояние "Z" (зомби).

PID (Process ID) — это уникальный идентификатор процесса в операционной системе. Каждый процесс, запущенный в системе, получает свой собственный PID, который используется для его идентификации и управления. PID позволяет операционной системе различать и управлять множеством процессов, работающих одновременно.
Основные характеристики PID:

    Уникальность: В каждый момент времени PID уникален для каждого процесса в системе. Когда процесс завершается, его PID может быть повторно использован для нового процесса.

    Последовательность: PID обычно присваиваются последовательно, начиная с некоторого базового значения (например, 1 для процесса init в Linux).

    Ограниченность: Диапазон возможных значений PID ограничен. В Linux, например, максимальное значение PID по умолчанию равно 32768, но его можно изменить в зависимости от конфигурации системы.

    Родительские и дочерние процессы: Процессы могут порождать другие процессы (дочерние процессы), и каждый процесс имеет родительский процесс, кроме процесса init, который является корневым процессом.
